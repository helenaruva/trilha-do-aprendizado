Tipos de software e controles de versão

Centralizado

Possui um repositório central em um servidor com controle de versões.

Distribuído

É independente do servidor, as atualizações são feitas no repositório central. Possui um repositório local com controle de versões.
Como por exemplo o GIT, que é um software livre e de versionamento criado em 2005. Nele é mostrado o histórico completo
do código para revisão e também indende de acesso as redes.

Tipos de comando GIT

  • fetch - Confere se seu repositório está sincronizado.
  • push - Envia as aterações para o servidor.
  • pull - Sincroniza o repositório.
  • commit - Torna permanente as alterações no repositório.

    • Branches

    Os branches servem para o progamador fazer atualizações no código sem afetar o código principal e quem esteja usando ele. São como um universo paralelo
    ao código principal.

    No Github Desktop, você deve criar um branch e configurá-lo, nele haverá tudo que já existe no código principal.

    As alterações do código devem ser feitas na branch para depois de prontas e testadas serem lançadas no código principal após o PUSH.
